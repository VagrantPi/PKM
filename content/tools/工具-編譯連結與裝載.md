## 🎯 什麼情境該想到我
當你遇到「編譯過但連結失敗、undefined symbol、找不到入口」這類錯誤，或想搞懂「原始碼怎麼變成能跑的程式」時。

## ⚙️ 怎麼用（理解四階段，對症下藥）
1. **預處理**：展開巨集、include（問題：標頭/巨集展開錯）。
2. **編譯**：每個 .c/.cpp → 目的檔（.o）；此階段錯＝語法/型別問題。
3. **連結**：把多個目的檔 + 庫的符號兜起來、做重定位。**undefined symbol = 有引用、沒定義**（少連某個 .o/庫，或簽名不符）。
4. **裝載**：作業系統把執行檔映射進記憶體、解析動態庫、跳到進入點執行。**執行期找不到 .so/.dll = 動態庫路徑/版本問題**。

依錯誤發生在哪一階段，就知道往哪查。
