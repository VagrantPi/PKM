## 🎯 什麼情境該想到我
當工程師與業務/PM「講的是同一件事卻用不同詞」、需求翻成程式碼常失真時。

## ⚙️ 怎麼用
1. **建立一套團隊 + 領域專家共用的詞彙**：一個概念一個名字，全員（含文件、對話、程式碼）都用它。
2. **讓程式碼直接反映這套語言**：類別/方法名 = 領域術語，讀 code 像讀業務。
3. **發現詞彙模糊或衝突，就當場澄清並更新**——語言的演進就是模型的演進。
4. 詞彙若在不同脈絡意義不同 → 那是要切 [[工具-限界上下文]] 的訊號。
