## 🎯 什麼情境該想到我
當你的程式碼被一堆「檢查回傳值/錯誤碼」的 if 淹沒，或常被 null 炸到時。

## ⚙️ 怎麼用
1. **用例外（Exception）而非回傳錯誤碼**：讓正常流程乾淨，錯誤處理分離。
2. **別回傳 null，也別傳 null 當參數**：改回傳空集合、Optional，或丟例外。
3. **例外訊息要有情境**：說清楚哪裡、為什麼失敗。
4. **把 try/catch 的主體抽成獨立函式**，讓錯誤處理成為「一件事」。
