## 🎯 什麼情境該想到我
當你要把資料/請求分散到多台節點（快取、分片 DB），又希望「增減節點時不要大搬風」時。

## ⚙️ 怎麼用
1. **問題**：用「hash(key) % N」分配，一旦 N 改變（加/減機器），幾乎所有 key 都要重新對應 → 快取全失效、大量資料搬移。
2. **一致性雜湊**：把節點與 key 都雜湊到一個環（0 ~ 2³²）上；key 順時針找到第一個節點。
3. **增減節點時，只影響環上相鄰的一小段 key**，其餘不動 → 資料搬移量最小化。
4. **虛擬節點（vnodes）**：每個實體節點在環上放多個虛擬點，讓分佈更均勻、避免熱點。
