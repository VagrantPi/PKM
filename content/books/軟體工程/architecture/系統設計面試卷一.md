## 📌 30 秒摘要（Layer 3）
> 系統設計面試系列的第一卷。前半奠定通用能力——**如何從 0 擴展到百萬用戶**、四步設計框架、粗估容量（這些與 [[系統設計面試]] 卷二共用，見既有工具）；後半是一系列經典設計案例（限流器、一致性雜湊、鍵值儲存、唯一 ID、短網址、爬蟲、通知、動態訊息、聊天、搜尋自動補全…），每個案例都沉澱出可重用的**設計積木**。

## ✨ 關鍵重點（Layer 1–2）
- **從 0 到百萬用戶的擴展路徑**：單機 → 分離 DB → 負載均衡 → 快取/CDN → 讀寫分離 → 分片。
- **反覆出現的積木**：一致性雜湊（資料分佈）、限流（保護）、唯一 ID 生成、複製與分片、訊息佇列。
- **案例導向**：每題示範「釐清需求 → 高層設計 → 深入 → 取捨」的完整流程。

## 💬 金句原文（Layer 0）
- 「好的系統設計，是在明確的約束下做出合理的取捨。」
